# backend — API Service

HTTP API backing the desktop client. **Go + chi + GORM + PostgreSQL + JWT.** Deployed to Kubernetes at `cac.guz-studio.dev`.

## Responsibilities

- **Auth** — username/password login → JWT access (60m) + refresh (24h) tokens.
- **Servers** — register/list/delete user-owned VPSes. SSH private keys stored in the host OS keychain (not the DB).
- **Collections** — Postman-style HTTP request trees with folder/request nodes. Owner-based + share-with-user model.
- **Users** — search endpoint for collection sharing.

## Endpoints

All routes are prefixed `/api/v1`. **List/create routes require a trailing slash** because chi mounts `r.Get("/", ...)` as `/<resource>/`.

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/health` | — | Liveness probe |
| `POST` | `/api/v1/auth/login` | — | username/password → token pair |
| `POST` | `/api/v1/auth/refresh` | refresh | New access token |
| `GET` | `/api/v1/auth/me` | access | Current user session |
| `GET` | `/api/v1/users/search` | access | Search users (for sharing) |
| `GET` | `/api/v1/servers/` | access | List user's servers |
| `POST` | `/api/v1/servers/` | access | Register server (SSH key → keychain) |
| `DELETE` | `/api/v1/servers/{id}` | access | Remove server |
| `POST` | `/api/v1/servers/{id}/deploy-agent` | access | Install `swarm-manage` on host |
| `POST` | `/api/v1/servers/{id}/update-agent` | access | Update `swarm-manage` on host |
| `GET` | `/api/v1/collections/` | access | List owned + shared |
| `POST` | `/api/v1/collections/` | access | Create |
| `GET` | `/api/v1/collections/{id}` | access | Detail + node tree |
| `PUT` | `/api/v1/collections/{id}` | access | Rename / update metadata |
| `DELETE` | `/api/v1/collections/{id}` | access | Delete (cascades to nodes/shares) |
| `PUT` | `/api/v1/collections/{id}/tree` | access | Replace full node tree |
| `GET` | `/api/v1/collections/{id}/shares` | access | List recipients |
| `POST` | `/api/v1/collections/{id}/shares` | access | Share with user |
| `DELETE` | `/api/v1/collections/{id}/shares/{userId}` | access | Revoke share |

## Environment variables

See `.env.example`. Required:

- `DATABASE_URL` — full Postgres DSN (`host=... user=... password=... dbname=... port=5432 sslmode=disable`)
- `JWT_SECRET_ACCESS` / `JWT_SECRET_REFRESH` — HMAC secrets
- `ADMIN_USERNAME` / `ADMIN_PASSWORD` — seeded on first boot if no users exist
- `PORT` — defaults to `8080`

## Local development

```bash
cp .env.example .env             # fill in DATABASE_URL + JWT secrets
air                              # hot reload via .air.toml
```

The migration is auto-run on boot via GORM `AutoMigrate` in `repository/db.go`.

## Build & deploy

`.github/workflows/backend.yml` triggers on push to `backend/**` or manual `workflow_dispatch`:

1. `docker build` → push to `ghcr.io/romanshkvolkov/c-c/backend:latest`
2. SSH to k8s host → `kubectl create secret` (from GitHub repo secrets) → `kubectl apply -f k8s/` → `rollout status`

> **Operational note:** when you rotate the DB password, **also update the `DATABASE_URL` GitHub repo secret** before the next deploy. Otherwise the pod crash-loops on a dead credential and the API returns `503` upstream errors.
